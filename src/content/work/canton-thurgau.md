## The Context

The Canton of Thurgau needed an internal IT service portal — a system for employees to order services, submit tickets, and track requests. Government systems live a long time and get used by a wide range of people. Getting the UX and the architecture right mattered.

KiloKilo built it. I led the frontend development.

## What We Built

**The service portal:**
A TypeScript and Next.js application that handles the full service lifecycle — browsing the service catalogue, submitting requests, approval routing, and status tracking. Designed for a non-technical internal audience.

**The component library:**
Not just a set of UI components, but a design system — built to be reusable, documented, and maintainable by a team that would extend it over time. Government projects don't get rewritten every two years.

## The Constraints

Public sector work has its own set of constraints: accessibility requirements, procurement processes, stakeholders who aren't product people, and systems that have to work reliably for years without significant investment.

The architecture had to be straightforward. The components had to be accessible. The documentation had to be good enough for a developer who wasn't in the room when the decisions were made.

## Outcome

A live internal service portal deployed to the Canton of Thurgau. A component library that outlasts the project. The kind of work that's invisible when it's done right.
